# syntax_python

Репозиторий с практическими заданиями по синтаксису Python. Содержит разобранные задачи по темам: вывод данных, арифметика, ветвление, циклы, работа с цифрами числа и логические выражения и тд.

---

## Часть 1 — Как получить этот репозиторий себе

### Вариант А: Просто скачать (без Git, без регистрации)

1. Перейди на страницу репозитория: `https://github.com/MaximBytecamp/syntax_python`
2. Нажми зелёную кнопку **`<> Code`**
3. Выбери **`Download ZIP`**
4. Распакуй архив в любую папку — готово

---

### Вариант Б: Клонировать через Git (рекомендуется)

> Нужен установленный Git: https://git-scm.com/downloads

**Шаг 1.** Открой терминал (Terminal на macOS/Linux, Git Bash или PowerShell на Windows)

**Шаг 2.** Перейди в папку, куда хочешь скачать проект:
```bash
cd ~/Desktop
```

**Шаг 3.** Клонируй репозиторий:
```bash
git clone https://github.com/MaximBytecamp/syntax_python.git
```

**Шаг 4.** Зайди в папку:
```bash
cd syntax_python
```

Всё — файлы у тебя на компьютере. Чтобы получить последние обновления в будущем:
```bash
git pull
```

---

### Вариант В: Сделать форк (Fork) — своя копия на GitHub

Форк — это личная копия чужого репозитория у тебя на GitHub. Ты можешь менять её как угодно, не затрагивая оригинал.

**Шаг 1.** Войди в свой аккаунт на [github.com](https://github.com)

**Шаг 2.** Перейди на страницу: `https://github.com/MaximBytecamp/syntax_python`

**Шаг 3.** Нажми кнопку **`Fork`** (в правом верхнем углу страницы)

**Шаг 4.** GitHub создаст копию репозитория у тебя в аккаунте: `https://github.com/ТВО_ИМЯ/syntax_python`

**Шаг 5.** Теперь клонируй уже свою копию:
```bash
git clone https://github.com/ТВО_ИМЯ/syntax_python.git
```

Теперь ты можешь вносить изменения, коммитить и пушить — всё это будет только в твоём форке.

## Часть 2 — Как создать свой репозиторий и заливать в него код

### Шаг 1. Зарегистрируйся на GitHub

Если ещё нет аккаунта — зайди на [github.com](https://github.com) и пройди регистрацию.

---

### Шаг 2. Установи Git

- **macOS:** `brew install git` или скачай с https://git-scm.com
- **Windows:** скачай установщик с https://git-scm.com/download/win
- **Linux:** `sudo apt install git`

Проверь, что Git установлен:
```bash
git --version
```

---

### Шаг 3. Представься Git (один раз)

```bash
git config --global user.name "Твоё Имя"
git config --global user.email "твой@email.com"
```

> Это нужно, чтобы коммиты подписывались твоим именем.

---

### Шаг 4. Создай репозиторий на GitHub

1. Нажми **`+`** → **`New repository`** (правый верхний угол)
2. Введи название (например: `my_python_practice`)
3. Выбери **Public** (виден всем) или **Private** (только тебе)
4. Можно поставить галочку **`Add a README file`** (НО НЕ РЕКОМЕНДУЕТСЯ)
5. Нажми **`Create repository`**

---

### Шаг 5. Свяжи локальную папку с GitHub

**Вариант А — если папка уже есть на компьютере:**

```bash
cd /путь/до/твоей/папки

git init                                # инициализировать Git в папке
git add .                               # добавить все файлы в индекс
git commit -m "первый коммит"           # создать первый коммит
git branch -M main                      # назвать ветку main
git remote add origin https://github.com/ТВО_ИМЯ/my_python_practice.git
git push -u origin main                 # залить на GitHub
```

**Вариант Б — если хочешь начать с нуля:**

```bash
git clone https://github.com/ТВО_ИМЯ/my_python_practice.git
cd my_python_practice
# создавай файлы здесь
```

---

### Шаг 6. Ежедневная работа — цикл из трёх команд

После того как ты написал код и хочешь сохранить его на GitHub:

```bash
git add .                               # добавить все изменённые файлы
git commit -m "описание изменений"      # сохранить снимок (коммит)
git push                                # отправить на GitHub
```

> **Хорошее правило:** пиши осмысленные сообщения к коммитам.
>
> ❌ Плохо: `git commit -m "fix"`
>
> ✅ Хорошо: `git commit -m "задание 6.46 — анализ цифр числа"`

---

### Шаг 7. Посмотреть статус и историю

```bash
git status          # что изменилось и что ещё не добавлено
git log --oneline   # краткая история коммитов
git diff            # посмотреть изменения построчно
```
